01 ✦ Brand Assets

Brand Assets Database

A dedicated brand assets page built within each brand's existing website, all logos, print files, digital assets, and press kits, browsable and downloadable. Launching with Fina, then scaling across the rest.

Execution Plan
  • 1
    Audit and organize all existing Fina assets, logos, color palettes, typography files, icons, imagery
  • 2
    Design and build a brand assets page within Fina's existing website as a new dedicated section
  • 3
    Define access tiers, open download for standard assets, request-based for premium press kits
  • 4
    Establish file format standards and naming conventions across all asset types
  • 5
    Validate with Fina, then build the same system for all the other brands

02 ✦ Brand Guidelines

Brand Guidelines Online

Our brands have guidelines but lack depth, missing UI direction, brand architecture, and the detail seen in world-class brands. Building a fully detailed, living guidelines page within each brand's website. Starting with Fina, then scaling across the rest.

What Will Be Covered
Logo System & Usage Color & Typography Brand Architecture Photography Direction Motion & Animation UI Component Guidelines Tone of Voice Do's & Don'ts
Execution Plan
  • 1
    Conduct a brand audit for Fina: identify what exists and what's missing
  • 2
    Design and develop the guidelines page within Fina's website (e.g. fina.com/brand)
  • 3
    Populate all sections with documentation, visual examples, and interactive references
  • 4
    Publish and make accessible via a direct link, internally and externally
  • 5
    Roll out to the remaining brands, each living on their respective website

04 ✦ Design System

Automated Design System Portal

An internal AI-powered portal where any team member inputs content and receives a brand-compliant creative. Powered by the Codex API as brain, trained per brand on their specific guidelines.

Execution Plan
  • 1
    Design and build the portal frontend, prompt input + live creative preview panel
  • 2
    Build backend on Claude; wire Codex API as the generative brain
  • 3
    Feed each brand's guidelines as training context, one profile per brand
  • 4
    Host on a dedicated server and deploy for org-wide access
Stack
Codex API Brain Claude Backend Brand-Trained Outputs Prompt Refinement Multi-Brand Support Dedicated Server
What makes it different
The Codex API brain is trained on each brand's visual language specifically. A Fina output thinks in Fina's identity, not a generic template.

05 ✦ Company Profile

Company Profile Automation

A web-based tool where authorized users input detailed company data and the software populates a beautifully designed, presentation-style company profile, ready to export as PDF or print. Admin input is separate from the public-facing styled output.

Execution Plan
  • 1
    Build a working prototype, detailed admin input and designed deck output views
  • 2
    Finalize design and UX through stakeholder review
  • 3
    Hand off to the dev team for full deployment
Two-Sided Product
Admin Backend
Password-protected, limited access. Detailed input fields for all business data, hubs, financials, investors, milestones, services, and more.
Public Frontend
Polished, deck-style output pages. Users select which sections to include, hit Export PDF or Print, no design work needed.
